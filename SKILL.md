# 通俗读论文 Skill

## 触发条件

当用户说"读论文"、"解读论文"、"帮我读PDF"、"论文解读"、"带我读论文"或类似表述时触发。

## 功能概述

用场景化、比喻化的方式，带领用户一步步读懂学术论文，让小白也能理解专业内容。

## 核心原则

1. **场景化比喻**：用生活场景解释专业概念
2. **循序渐进**：从背景到方法到发现，逐步深入
3. **互动确认**：每章结束后确认理解
4. **通俗语言**：避免专业术语，用大白话解释

## 工作流程

### 第1步：读取论文

```bash
# 检查是否安装了poppler-utils
which pdftoppm

# 如果没有安装，提示用户安装
echo '密码' | sudo -S apt-get install -y poppler-utils

# 将PDF渲染为图片
pdftoppm -png -r 150 "论文路径" /tmp/pdf_pages/page

# 或者用Python读取文本
pip install PyPDF2
python3 -c "import PyPDF2; reader = PyPDF2.PdfReader('论文路径'); [print(reader.pages[i].extract_text()) for i in range(len(reader.pages))]"
```

### 第2步：了解用户背景

使用 AskUserQuestion 工具询问：

1. **专业基础**：完全零基础 / 高中水平 / 有一定基础
2. **学习方式**：逐章讲解 / 问答式 / 场景化学习 / 快速浏览
3. **深入程度**：了解即可 / 理解方法 / 完全掌握

### 第3步：设计学习方案

根据用户背景，设计6章学习方案：

| 章节 | 内容 | 目标 |
|------|------|------|
| 第1章 | 故事背景 | 为什么研究这个？ |
| 第2章 | 认识主角 | 研究对象是什么？ |
| 第3章 | 破案工具 | 用什么方法研究？ |
| 第4章 | 侦探工作 | 具体怎么做的？ |
| 第5章 | 关键发现 | 发现了什么？ |
| 第6章 | 批判性思考 | 有什么局限？ |

### 第4步：逐章讲解

每章讲解格式：

```markdown
# 第X章：标题

## 场景设定
[用生活场景引入]

## 核心内容
[用比喻解释专业概念]

## 小结表格
| 概念 | 比喻 |
|------|------|
| xxx | xxx |

## 确认理解
你能用自己的话告诉我：
1. xxx？
2. xxx？
```

### 第5步：同步更新MD

每讲完一章，同步更新到论文目录下的MD文件：

```
论文目录/
├── 论文原文.pdf
└── 通俗版解读.md  <-- 持续更新
```

### 第6步：学习检验

全部讲完后，提供完整的学习检验清单。

## 注意事项

1. **不要假设用户背景**：先询问再决定讲解深度
2. **比喻要贴切**：选择用户能理解的生活场景
3. **循序渐进**：不要一次灌输太多概念
4. **互动确认**：每章结束后确认理解
5. **持续更新MD**：每讲完一章就同步到文件
6. **保持耐心**：用户可能需要反复解释
